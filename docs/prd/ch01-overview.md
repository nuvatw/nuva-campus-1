## 1.1 背景說明

目前網站是一個混合式的校園大使管理平台，所有功能都暴露在首頁上。需要重構為一個有明確身份區分的品牌互動區，讓四種不同角色能夠快速進入屬於自己的專區。

## 1.2 四種角色定義

| 角色 | 說明 | 專區功能 |
|------|------|----------|
| **努努** | 活動參與者 | 查看活動資訊、報名 |
| **校園大使** | 品牌大使 | 任務系統、工作坊、存活狀態 |
| **守護者** | 活動執行團隊 | 報到系統、便當領取 |
| **都不是** | 一般訪客 | 合作申請、訂閱最新動態 |

## 1.3 核心目標

- **身份區隔**：透過入口選擇器區分四種用戶類型
- **安全性**：密碼存入資料庫，可隨時修改
- **日系美學**：簡約、高級感的視覺設計
- **郵件通知**：一鍵發送報到編號給參與者
- **用戶體驗**：絲滑動畫、流暢載入、完美響應式

## 1.4 現有架構問題

| 問題 | 影響 |
|------|------|
| 所有內容暴露在首頁 | 不同角色無法快速定位 |
| 無身份驗證機制 | 任何人都能存取內部資料 |
| 缺少招募管道 | 無法吸引新合作夥伴 |
| 無執行團隊工具 | 活動現場報到效率低 |

## 1.5 Chain of Thought - 總體實施步驟

```
1. 設計日系簡約視覺系統
2. 建立 Supabase 密碼管理表
3. 規劃新的路由架構
4. 建立核心組件（入口選擇器、密碼驗證等）
5. 遷移現有功能到對應專區
6. 新增守護者專區功能
7. 實作郵件通知系統
8. 加入動畫效果
9. 響應式測試與優化
```
