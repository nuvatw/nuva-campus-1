## 7.1 功能描述

為一般訪客提供兩個表單區域，設計遵循日系簡約美學。

1. **合作單位申請**：想在學校舉辦活動的單位申請
2. **訂閱最新動態**：對校園巡迴計劃有興趣的人訂閱

## 7.3 表單欄位規格

### 合作申請表單

| 欄位 | 類型 | 必填 | 驗證規則 |
|------|------|------|----------|
| school | text | ✓ | 最少 2 字 |
| department | text | ✓ | 最少 2 字 |
| job_title | text | ✓ | 最少 2 字 |
| name | text | ✓ | 最少 2 字 |
| phone | tel | ✓ | 台灣手機格式 |
| email | email | ✓ | Email 格式 |
| message | textarea | ✗ | 最多 500 字 |

### 訂閱表單

| 欄位 | 類型 | 必填 | 驗證規則 |
|------|------|------|----------|
| name | text | ✓ | 最少 2 字 |
| email | email | ✓ | Email 格式、不可重複 |

## 7.5 Chain of Thought - 實施步驟

```
1. 建立表單組件
   - PartnershipForm.tsx（合作申請）
   - NewsletterForm.tsx（訂閱表單）

2. 建立表單驗證 Hook
   - useFormValidation
   - 支援各種驗證規則
   - 即時錯誤提示

3. 建立 /app/recruit/page.tsx
   - 頁面標題和介紹
   - 合作申請區塊
   - 訂閱表單區塊
   - 返回首頁按鈕

4. 連接 Supabase
   - 合作申請提交
   - 訂閱表單提交
   - 錯誤處理

5. 添加成功提示
   - 成功畫面過場
   - 表單重置
```
